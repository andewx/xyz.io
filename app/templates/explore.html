Core Modular Framework

Everything you need to know to start extending xyz.io

ModelKit

Modelkit ModelObject class objects are self-replication JSON stores for storing all of your application model. When you add a new model class you can automatically integrate into the database. It also gets automatically handles by the REST CRUD framework for Server API model handling.

DBKit

The xyz.dbkit class DBMain handles model storage and query operations. When you store a model and add it to ModelKeys a node is automatically created for that class where default CRUD storage occurs. DBKit stores objects in a native JSON format and auto-syncs graph writing in a background thread.

Webkit

The Webkit module handles HTML Templating. You can load in html templates and decorate the document with @:somekey keys. Replace these keys to easily fill in content as you load and send to the browser.

Route Registration

xyz.io has one main dependency, Javalin.io for implementing HTTP handlers. Add your route names to the RouteManager along with the handler and xyz.io does the rest for you.

Notifications

Xyz.io ships with a notification database tasking that sync in its own thread. Notifications simply deposit messages from one object to another. Once a message has been read it is cleaned up by the database.

Site Design

You can design your own sites in Xyz.io in the admin dashboard. Simply create a site and add in your own templates/css to begin.

XYZ.io is a multiplatform solution. See how to integrate into these platforms.
